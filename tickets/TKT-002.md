## Description

The posts/blog rendering pipeline shipped in TKT-001, but there's no way to discover it from the site. Other page types are documented in `page-types.mdx` (guide template, YAML examples, links to live demos). Posts needs the same treatment.

Two things are missing:

1. A showcase/documentation page explaining the posts feature, matching the style of `page-types.mdx` -- guide template, frontmatter reference, usage examples, link to the live `/posts/` listing as a demo.
2. A "Posts" link in the site nav (`site.config.ts`).

## What to build

- `src/content/pages/posts-guide.mdx` -- Documentation page for the posts feature. Use the `guide` template. Cover:
  - What posts are for (date-driven chronological content: blog, news, updates)
  - Required and optional frontmatter fields with YAML code block examples
  - How drafts work (`draft: true` filters from listing and individual routes)
  - Reading time (auto-calculated from body length)
  - Tags (displayed on cards and individual posts)
  - Link to `/posts/` as a live example
  - Use `Callout` components where appropriate (match existing doc style)

- Update `src/site.config.ts` -- Add `{ label: "Posts", href: "/posts" }` to the nav array in a sensible position (after existing page types or at the end).

- Add 1-2 more sample posts to `src/content/posts/` so the listing page looks less empty in the demo. Short, varied content (different tags, different lengths for varied reading times). These are framework demo content, not site-specific.

## Constraints

- Match the documentation style of `page-types.mdx` exactly -- guide template, structured headings, YAML code blocks, Callout components, links to live examples.
- No em dashes in copy.
- Framework files, not site-specific.
- Keep sample posts short and generic.

## Acceptance Criteria
- [ ] `src/content/pages/posts-guide.mdx` exists with guide template, documents all posts frontmatter fields
- [ ] `site.config.ts` nav includes a "Posts" link
- [ ] 1-2 additional sample posts exist in `src/content/posts/`
- [ ] `/posts/` listing shows multiple posts
- [ ] `npm run build` passes
