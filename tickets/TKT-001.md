## Description

The posts content collection schema already exists in `src/content.config.ts` (lines 52-67) and the mobile editor already supports creating/editing posts. What's missing is the rendering layer: there's no way to actually view posts on a Loomwork site.

Create these framework files:

- `src/pages/posts/[slug].astro` - Individual post route. Query the posts collection (filter drafts). Render using the existing `Content.astro` layout. Display date, author, tags, and reading time. Similar pattern to `src/pages/[...slug].astro` but for the posts collection.

- `src/pages/posts/index.astro` - Post listing page. Reverse-chronological. Query posts collection, filter drafts, sort by date descending. Use `Base.astro` layout. Show a card per post with title, date, description, tags, and reading time estimate.

- `src/components/PostCard.astro` - Card component used by the listing page. Props: title, date, description, tags, slug, body (for reading time calc). Links to `/posts/{slug}`.

## Constraints

- These are framework files, not site files. They should work generically for any Loomwork site.
- Use the existing `Content.astro` layout for individual posts (template "default"). Don't create a new layout.
- Match the existing code style in the repo (check `[...slug].astro`, `Content.astro`, and any component like `Callout.astro` for patterns).
- The listing page should read `SITE.name` from `site.config.ts` for the page title (e.g. "Posts | {SITE.name}").
- No em dashes in any copy.
- Keep it minimal. No tag archive pages, no RSS, no author pages. Just the listing and individual post routes. Those can come later.

## Acceptance Criteria
- [ ] `src/pages/posts/[slug].astro` renders individual posts from the posts collection
- [ ] `src/pages/posts/index.astro` lists posts in reverse-chronological order
- [ ] `src/components/PostCard.astro` card component works on the listing page
- [ ] Drafts are filtered out in both routes
- [ ] Reading time is displayed on both the listing and individual post pages
- [ ] A sample post at `src/content/posts/hello-world.mdx` exists with valid frontmatter
- [ ] `npm run build` completes with no errors

### Posts collection schema (`src/content.config.ts`, lines 52-66)

The posts collection is already defined with these frontmatter fields:
- `title` (string, required)
- `description` (string, max 160 chars, required)
- `date` (coerced Date, required)
- `author` (string, default "Dan")
- `tags` (string array, default [])
- `hero_image` (string, optional)
- `hero_alt` (string, optional)
- `draft` (boolean, default false)
- `og_image` (string, optional)

The collection is exported at line 68: `export const collections = { pages, posts }`.

### Existing page routing (`src/pages/[...slug].astro`)

The catch-all page route follows this pattern:
1. `getStaticPaths()` calls `getCollection("pages", ({ data }) => !data.draft)` to filter drafts at build time.
2. Maps entries to `{ params: { slug: page.id.replace(/\.mdx?$/, "") }, props: { page } }`.
3. Calls `page.render()` to get `Content` (rendered MDX) and `headings`.
4. Passes props to the `Content.astro` layout including `title`, `description`, `template`, `hero_image`, `hero_alt`, `date_updated`, `headings`, and `body`.

### Content.astro layout (`src/layouts/Content.astro`)

- Wraps `Base.astro`, accepts Props: `title`, `description`, `template`, `hero_image`, `hero_alt`, `date_updated`, `headings`, `body`.
- Computes reading time from `body` at line 29: `body.trim().split(/\s+/).length` / 230 wpm.
- Displays reading time and date in `.page__meta`.
- Has existing CSS for `.page`, `.page__header`, `.page__hero`, `.page__content.prose`.
- No `author` or `tags` props currently -- the post route will need to add those to the header manually or pass them in a slightly different way.

### Base.astro layout (`src/layouts/Base.astro`)

- Accepts `title`, `description`, `og_image`, `noindex`, `class`.
- Computes `pageTitle` at line 30: `title === SITE.name ? title : \`${title} - ${SITE.name}\``.
- Imports `SITE` from `../site.config` (line 9). The listing page should use `SITE.name` via the same import.
- Renders `<Header />`, `<main>`, `<Footer />`, plus optional reader controls.

### Site config (`src/site.config.ts`)

- `SITE.name` = "Loomwork". The listing page title should be `Posts | ${SITE.name}`.

### Component style reference (`src/components/Callout.astro`)

- Uses typed `interface Props` in frontmatter.
- CSS uses `var(--space-*)`, `var(--radius)` design tokens.
- Supports dark mode via `:global([data-dark="true"])`.

- `src/content/posts/` exists but is empty (no sample post).
- No `PostCard.astro` component exists.

## Proposed Solution
<!-- Agent fills this during /ticket-investigate -->

Create three new framework files plus one sample content file:

1. **`src/pages/posts/[slug].astro`** -- Individual post route. Follows the same `getStaticPaths` + `getCollection` pattern as `[...slug].astro` but queries the `posts` collection. Renders via `Content.astro` layout (template "default"). Adds post-specific metadata (date, author, tags) in a header block above the rendered content.

2. **`src/pages/posts/index.astro`** -- Listing page. Uses `Base.astro` directly (like a landing page, not Content.astro). Queries posts collection, filters drafts, sorts by `date` descending. Renders a `PostCard` for each post. Page title: `Posts | ${SITE.name}`.

3. **`src/components/PostCard.astro`** -- Presentational card component. Accepts `title`, `date`, `description`, `tags`, `slug`, `body` (for reading time). Renders as a linked card with title, date, description, tags, and reading time. Uses existing design tokens for styling. Supports dark mode.

4. **`src/content/posts/hello-world.mdx`** -- Sample post with valid frontmatter to satisfy acceptance criteria and let the build succeed (Astro requires at least one entry in a content collection directory for type generation).

The individual post page will pass `body` to `Content.astro` for its built-in reading time calculation. For the listing page, reading time will be calculated in `PostCard.astro` using the same formula (word count / 230).

Post-specific metadata (date, author, tags) will be rendered in the `[slug].astro` page itself, inserted before the `<Content>` layout's slot content via a wrapper div, or by extending the Content.astro header. The cleanest approach: render via `Content.astro` and add a post-meta block between the header and content using a named slot or by placing it inside the default slot before `<RenderedContent />`.

Looking at Content.astro more carefully: the `<slot />` is where rendered MDX goes. The post metadata (date, author, tags) should appear in the page header area. Since Content.astro already renders a header with title + reading time + date_updated, the post route can:
- Pass `date_updated` as the post's `date` so the date appears in the existing meta area.
- Add author and tags as additional content inside the slot, before the MDX.

Actually, a cleaner approach: wrap the rendered content in the post route file itself, placing post metadata (author, tags, formatted date) inside the slot passed to Content.astro, right before `<RenderedContent />`. This avoids modifying Content.astro's interface.

- [ ] 1. Create `src/pages/posts/[slug].astro` -- Individual post route.
  - Import `getCollection` from `astro:content` and `Content` layout from `../../layouts/Content.astro`.
  - Define `getStaticPaths()`: call `getCollection("posts", ({ data }) => !data.draft)`, map entries to `{ params: { slug: entry.id.replace(/\.mdx?$/, "") }, props: { post: entry } }`.
  - In the component script: destructure `post` from `Astro.props`, call `post.render()` to get `Content` (as `RenderedContent`) and `headings`.
  - Compute `readingTime` with the same formula as Content.astro: `Math.max(1, Math.round(post.body.trim().split(/\s+/).length / 230))`.
  - Render `<Content>` layout with `title={post.data.title}`, `description={post.data.description}`, `template="default"`, `hero_image={post.data.hero_image}`, `hero_alt={post.data.hero_alt}`, `date_updated={post.data.date}`, `headings={headings}`, `body={post.body}`.
  - Inside the `<Content>` slot: render a `.post-meta` div with author, formatted date, and tag pills, followed by `<RenderedContent />`.
  - Add scoped `<style>` for `.post-meta` (flex row, gap, tag pills using design tokens, dark mode support).

- [ ] 2. Create `src/components/PostCard.astro` -- Card component for listing page.
  - Define `interface Props` with `title: string`, `date: Date`, `description: string`, `tags: string[]`, `slug: string`, `body: string`.
  - Compute reading time: `Math.max(1, Math.round(body.trim().split(/\s+/).length / 230))`.
  - Format date using `date.toLocaleDateString("en-US", { year: "numeric", month: "long", day: "numeric" })`.
  - Render: an `<a>` wrapping a `<article class="post-card">` linking to `/posts/${slug}`. Inside: `<h2>` for title, `<div class="post-card__meta">` with date + reading time + middot separator, `<p>` for description, tag pills div.
  - Add scoped `<style>` using design tokens (`--space-*`, `--radius`, `--color-border`, `--color-text-muted`). Include dark mode styles via `:global([data-dark="true"])`. Style the card with bottom border, hover state for the link.

- [ ] 3. Create `src/pages/posts/index.astro` -- Post listing page.

  - Query: `const posts = (await getCollection("posts", ({ data }) => !data.draft)).sort((a, b) => b.data.date.valueOf() - a.data.date.valueOf())`.
  - Render `<Base title={\`Posts | ${SITE.name}\`}>` wrapping a `.container` div.
  - Inside container: `<header>` with `<h1>Posts</h1>`, then a `.post-list` div mapping over posts, rendering `<PostCard>` for each with `title={post.data.title}`, `date={post.data.date}`, `description={post.data.description}`, `tags={post.data.tags}`, `slug={post.id.replace(/\.mdx?$/, "")}`, `body={post.body}`.
  - Handle empty state: if no posts, show a `<p>` saying "No posts yet."
  - Add scoped `<style>` for `.post-list` (flex column, gap) and header spacing.

- [ ] 4. Create `src/content/posts/hello-world.mdx` -- Sample post.
  - Frontmatter: `title: "Hello World"`, `description: "A first post to welcome readers and test the posts collection."`, `date: 2026-04-13`, `author: "Dan"`, `tags: ["welcome"]`, `draft: false`.
  - Body: a short paragraph (2-3 sentences) welcoming readers.

- [ ] 5. Run `npm run build` to verify no errors. Fix any type or rendering issues.
